# Overview

Capture 想做什么：
1. 工作中突然产生的随机想法：一个工具可以通过TUI或者CLI来记录想到的一些可以做的事情变成TO-DO-List
2. 随便什么时候突然产生的随机想法： 一个工具可以通过飞书/微信Bot发信息的方式记录下这个随机想法，方便写文章也好还是做一个小工具也好，但是需要记录下这个随机想法的来源，和当时的场景，哪怕是一些只言片语都行
3. 这些随机想法需要通过Capture工具记录成本地文件，后续需要执行就可以执行这些文档的意图性的东西了
4. 可以通过飞书CLI或者其他CLI，可以让openclaw这样的机器人随机接受这些随机想法，或者一个Bot可以接受这些信息，并且记录下来到本地
5. 可以通过Capture工具来执行这些随机想法，或者通过openclaw这样的机器人来执行这些随机想法
6. Capture工具还需要有一个看板，可以时刻看到这些任务的运行情况，是否完成等等
7. Capture工具可以通过飞书/微信Bot来通知用户这些任务的运行情况，是否完成等等
8. Capture工具可以配置调用什么样的AI Agent比如Claude Code 还是Codex 配合那个模型来执行这些任务
9. Capture 工具哪些做哪些不做可以通过Kanban操作，或者表格操作进行
10. Catpure工具的任务状态可以和飞书多维表格同步

请根据这些内容，设计一个Capture工具的PRD，包括功能点，非功能点，接口，数据库设计等。
请根据这些内容设计技术方案，计划，任务，任务优先级等等。
